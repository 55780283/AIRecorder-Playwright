# 实时录制日志功能

## ✨ 功能说明

现在 AI Recorder 支持**实时显示录制过程**，你可以在 popup 中看到每一个录制的操作步骤！

## 🎯 使用方法

### 1. 开始录制

```
1. 点击 AI Recorder 图标打开 popup
2. 输入 SOP 名称
3. 点击"开始录制"
4. 录制日志面板会自动显示
```

### 2. 查看实时日志

**录制过程中**：
- 📝 录制日志面板会显示在 popup 中
- 🔢 右上角显示已录制的操作数量
- ⏱️ 每个操作显示精确到秒的时间戳
- 🎨 不同类型的操作有不同的颜色标识

### 3. 操作类型和颜色

| 操作类型 | 图标 | 颜色 | 说明 |
|---------|------|------|------|
| 🌐 导航 | 🌐 | 蓝色 | 页面跳转、URL 变化 |
| 🖱️ 点击 | 🖱️ | 绿色 | 鼠标点击、按钮点击 |
| ⌨️ 输入 | ⌨️ | 橙色 | 文本输入、填写表单 |
| 📋 选择 | 📋 | 紫色 | 下拉菜单选择 |
| 📜 滚动 | 📜 | 粉色 | 页面滚动 |
| ⌨️ 按键 | ⌨️ | 青色 | 键盘按键（Enter、Tab 等） |

## 🔧 技术实现

### 更新机制

```javascript
// 每 500ms 检查一次录制状态
setInterval(async () => {
  if (this.isRecording) {
    await this.updateRecordingLog();
  }
}, 500);
```

### 智能更新

- ✅ 只在操作数量变化时才重新渲染
- ✅ 避免不必要的 DOM 操作
- ✅ 性能优化

### 状态同步

```javascript
// 从 background 获取最新的操作列表
const response = await chrome.runtime.sendMessage({ 
  type: 'GET_RECORDING_STATE' 
});
this.currentActions = response.session.actions;
```

## ⚙️ 配置选项

### 更新频率

默认每 **500ms** 更新一次日志。

如需修改，在 `popup.js` 中调整：

```javascript
// 修改更新间隔（单位：毫秒）
this.logUpdateInterval = setInterval(async () => {
  // ...
}, 500); // 改为其他值，如 1000 表示 1 秒
```

### 日志高度

默认最大高度 **200px**。

如需修改，在 `popup.css` 中调整：

```css
.recording-log {
  max-height: 200px; /* 改为其他值 */
}
```

## 🐛 故障排除

### 问题 1：日志不更新

**检查**：
1. 确认正在录制中
2. 检查浏览器控制台是否有错误

**解决**：
- 刷新 popup 页面
- 重新加载扩展

### 问题 2：日志显示为空

**检查**：
1. 确认已经执行了操作
2. 检查 content script 是否正常工作

**解决**：
- 在页面中执行一些操作
- 查看页面是否有录制指示器

### 问题 3：时间戳不准确

**原因**：
- 使用的是客户端时间
- 可能存在时区问题

**解决**：
- 确保系统时间正确
- 时间戳仅作为参考

## 📊 性能优化

### 1. 按需更新

只在操作数量变化时才更新 DOM：

```javascript
if (newActions.length !== this.currentActions.length) {
  this.currentActions = newActions;
  this.renderRecordingLog();
}
```

### 2. 限制日志数量

如果录制大量操作，可以考虑：

```javascript
// 只显示最近的 50 条记录
const recentActions = this.currentActions.slice(-50);
```

### 3. 防抖处理

避免频繁更新：

```javascript
// 使用防抖函数
const debouncedUpdate = debounce(() => {
  this.renderRecordingLog();
}, 300);
```

## 🎯 最佳实践

- 查看日志确认所有步骤都记录了
- 检查时间间隔是否合理
- 删除不必要的操作

# AI Recorder 使用说明

## ⚠️ 重要提示

### 录制状态说明

**Background Service Worker 是持久化的**，即使关闭 popup 页面，录制也会继续进行。

- ✅ **开始录制后**：可以关闭 popup 窗口，录制会继续
- ✅ **页面跳转**：录制状态会自动保持，不需要重新打开 popup
- ✅ **切换标签页**：录制状态会自动跟随
- ⚠️ **停止录制**：必须打开 popup 点击"停止录制"按钮

### 避免误录制

系统已经优化，**不会**在以下情况误录制：

- ❌ 页面刚加载时不会自动开始录制
- ❌ 没有点击"开始录制"按钮不会录制
- ❌ 刷新页面不会自动开始录制

## 📖 正确的使用流程

### 1. 开始录制

```
1. 点击浏览器工具栏的 AI Recorder 图标
2. 输入 SOP 名称（必填）
3. 输入描述（可选）
4. 点击"开始录制"按钮
```

**此时会发生**：
- ✅ Background Service Worker 开始录制会话
- ✅ 当前页面显示红色录制指示器
- ✅ 自动记录当前页面的导航信息

### 2. 执行操作

```
1. 可以关闭 popup 窗口（可选）
2. 在页面中执行需要录制的操作
3. 点击链接跳转到其他页面
4. 继续执行操作
```

**录制指示器**：
- 🔴 红色指示器始终显示在页面右上角
- 🔴 页面跳转后指示器会自动恢复
- 🔴 切换标签页指示器会跟随

### 3. 停止录制

```
1. 点击浏览器工具栏的 AI Recorder 图标
2. 点击"停止录制"按钮
3. 查看录制的 SOP 列表
```

**停止后**：
- ✅ 录制指示器消失
- ✅ SOP 被保存到本地存储
- ✅ 可以在 SOP 列表中查看和管理

## 🎯 常见使用场景

### 场景 1：单页面操作录制

**示例**：录制一个表单填写流程

```
1. 打开 popup，点击"开始录制"
2. 填写表单的各个字段
3. 点击提交按钮
4. 打开 popup，点击"停止录制"
```

### 场景 2：跨页面流程录制

**示例**：录制完整的登录流程

```
1. 打开 popup，点击"开始录制"
2. 点击"登录"链接 → 跳转到登录页
3. 输入用户名和密码
4. 点击登录按钮 → 跳转到仪表盘
5. 在仪表盘中执行操作
6. 打开 popup，点击"停止录制"
```

### 场景 3：长时间录制

**示例**：录制完整的购物流程

```
1. 打开 popup，点击"开始录制"
2. 关闭 popup（录制继续）
3. 浏览商品列表
4. 查看商品详情
5. 添加到购物车
6. 结算流程
7. 打开 popup，点击"停止录制"
```

## 🔍 查看和管理 SOP

### 查看 SOP 列表

```
1. 点击 AI Recorder 图标打开 popup
2. 在"SOP 列表"区域查看所有已保存的 SOP
3. 每个 SOP 显示：
   - SOP 名称
   - 步骤数量
   - 最后更新时间
```

### 查看 SOP 详情

```
1. 在 SOP 列表中点击"查看"按钮
2. 查看详细的操作步骤
3. 每个步骤显示：
   - 步骤序号
   - 操作类型（点击、输入、导航等）
   - CSS 选择器
   - 输入值（如果有）
```

### 导出 SOP

在 SOP 详情页面，可以导出三种格式：

1. **导出 Playwright** - TypeScript 测试脚本（.spec.ts）
2. **导出 JSON** - 结构化数据（.json）
3. **导出摘要** - Markdown 文档（.md）

### 删除 SOP

```
1. 在 SOP 列表中点击"删除"按钮
2. 确认删除
3. SOP 被永久删除
```

## ⚙️ 高级功能

### 录制状态持久化

**原理**：
- Background Service Worker 维护全局录制状态
- Content Script 在页面加载时查询录制状态
- 页面跳转后自动恢复录制状态和指示器

**优势**：
- ✅ 关闭 popup 不影响录制
- ✅ 页面跳转不中断录制
- ✅ 切换标签页继续录制

### 智能导航记录

**自动记录**：
- 页面加载时的初始导航
- 点击链接后的页面跳转
- 表单提交后的页面变化
- 浏览器前进/后退操作

### 防误触机制

**保护措施**：
- 页面加载延迟检查（500ms）
- 避免重复初始化
- 只在明确点击"开始录制"后才录制
- 背景页面不会自动开始录制

## 🐛 故障排除

### 问题 1：录制指示器不显示

**可能原因**：
- Content Script 未正确注入
- 页面是 chrome:// 内部页面

**解决方法**：
1. 刷新页面
2. 确保在 HTTP/HTTPS 页面
3. 检查浏览器控制台错误

### 问题 2：页面跳转后录制停止

**可能原因**：
- Background Service Worker 休眠
- 页面加载过慢

**解决方法**：
1. 等待页面完全加载
2. 检查 popup 中的录制状态
3. 如果已停止，重新开始录制

### 问题 3：无法停止录制

**可能原因**：
- Popup 页面未正确加载
- Background Service Worker 异常

**解决方法**：
1. 关闭并重新打开 popup
2. 在 chrome://extensions/ 重新加载扩展
3. 清除浏览器缓存

### 问题 4：SOP 列表为空

**可能原因**：
- 还未创建任何 SOP
- 存储数据被清除

**解决方法**：
1. 创建一个新的 SOP
2. 检查浏览器存储空间
3. 确保扩展有 storage 权限

## 💡 最佳实践

### 1. SOP 命名规范

```
✅ 推荐：
- "用户登录流程"
- "商品搜索和过滤"
- "购物车结算流程"

❌ 避免：
- "SOP 1"
- "测试"
- 空名称
```

### 2. 录制技巧

```
✅ 推荐：
- 录制前明确要录制的流程
- 使用稳定的页面元素
- 避免快速连续操作
- 每个 SOP 录制一个完整流程

❌ 避免：
- 录制无关的操作
- 使用动态生成的元素
- 页面未加载完成就操作
- 一个 SOP 包含多个独立流程
```

### 3. 选择器优化

系统会自动选择最优的选择器：

```
优先级：
1. element.id
2. data-testid, data-test, data-cy
3. aria-label
4. name, placeholder
5. CSS 路径
```

### 4. 录制时长控制

```
✅ 推荐时长：30 秒 - 5 分钟
- 太短：可能不完整
- 太长：难以维护

复杂流程可以分成多个 SOP
```
